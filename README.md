# 暗記カード PWA

スマートフォン・タブレット・PCで使えるスペースドリピティション暗記カードアプリです。

## フォルダ構成

```
/
├── index.html          ← アプリ本体
├── manifest.json       ← PWA 設定
├── sw.js               ← Service Worker（オフライン対応）
├── css/
│   └── style.css
├── js/
│   ├── db.js           ← IndexedDB 操作
│   ├── loader.js       ← データ取得・パース（DataSource抽象レイヤー）
│   ├── sm2.js          ← SM-2 アルゴリズム
│   ├── session.js      ← 学習セッション管理
│   ├── ui.js           ← UI ヘルパー
│   └── app.js          ← アプリ全体制御
├── data/
│   └── cards.xlsx      ← 問題データ（ここだけ編集すればOK）
└── images/
    ├── icon-192.png
    └── icon-512.png
```

---

## GitHub Pages へのアップロード手順

1. GitHub でリポジトリを作成（Public）
2. このフォルダの中身をすべてアップロード
3. Settings → Pages → Source を `main` ブランチ / `/ (root)` に設定
4. `https://ユーザー名.github.io/リポジトリ名/` でアクセス

**ローカルサーバー・Python 不要です。**

## 運用フロー

```
① PC で cards.xlsx を編集・保存
↓
② GitHub にアップロード（git push または Web UI）
↓
③ スマホでアプリを開く
↓
④ 起動時に自動で最新版を取得（ファイル選択不要）
↓
⑤ 学習開始
```

スマホでファイルを選択する操作は一切不要です。

## 問題データの編集

### ファイル
`data/cards.xlsx`（ファイル名は ASCII のみ。日本語不可）

### 列構成（1行目はヘッダー行）
| 列 | 内容 | 例 |
|----|------|----|
| A  | ID   | `s1`（科目内で一意） |
| B  | 単元 | `生命：植物のつくり` |
| C  | 問題 | `おしべの先にある袋を何というか。` |
| D  | 答え | `やく（葯）` |

### 科目を追加する
**シートを追加するだけ**でOKです。HTML/JS の編集は不要。

### 単元の表示名
`大分類：小分類` 形式にすると、単元フィルターで小分類名が表示されます。

---

## 起動時の動作

```
アプリ起動
↓
オンライン → data/cards.xlsx を fetch
├─ ETag 変化あり → DB 更新（SR 履歴は保持）→「更新しました」トースト
└─ ETag 変化なし → スキップ（何も表示しない）

オフライン or 取得失敗
└─ 保存済みデータを使用（トーストで通知）

DB にデータなし → エラー画面（「再試行」ボタン）
```

## 学習モード

| モード | 内容 |
|--------|------|
| 📅 今日の復習 | SM-2 で期限が来たカード |
| 📋 全問題 | 選択中の単元をランダム出題 |
| 💪 苦手問題 | 定着度（EF値）が低い順に出題 |
| 🎲 ランダム | 単元フィルタ対象をシャッフル |

## 将来 JSON に切り替える場合

`js/loader.js` の1行だけ変更すればOKです：

```js
// 現在（Excel）
const current = excel('data/cards.xlsx');

// JSON に切り替える場合
const current = json('data/cards.json');
```

JSON の形式：
```json
{
  "理科": [
    { "id": "s1", "unit": "生命：植物のつくり", "q": "問題文", "a": "答え" }
  ]
}
```

## トラブルシューティング

### 「問題データがありません」と表示される

1. `data/cards.xlsx` が正しいパスにアップロードされているか確認
2. ファイル名が `cards.xlsx`（ASCII）になっているか確認
3. 画面に表示されている「詳細」のエラーメッセージを確認
4. ブラウザの開発者ツール（コンソール）で `[boot]` ログを確認

### 問題データが更新されない

1. ホーム画面の「🔄 問題データを更新する」を押す
2. それでも反映されない場合は、ブラウザのキャッシュをクリア
